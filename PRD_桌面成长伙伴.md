# 产品需求文档（PRD）：桌面成长伙伴「晷 / Gnomon」

> 代号「晷」（gnomon，日晷的指针）——它丈量每一天的流逝，也记录你的成长。名称待定。

| 项目 | 内容 |
|---|---|
| 文档版本 | v0.2 |
| 状态 | 草稿，5 项关键决策已敲定 |
| 产品形态 | Windows 桌面应用（单机、个人使用） |
| 文档目的 | 定义产品范围、功能优先级与里程碑，作为开发依据 |

---

## 1. 背景与问题

使用者是一名研究生，日常面临三个反复出现的痛点：

1. **健忘**：任务（课程作业、待读文献、个人计划）一多就忘，缺少统一的收口。
2. **缺乏监督**：明知该每天读文献、写总结、做反思，但没有外部约束，容易拖延和断更。
3. **缺乏复盘闭环**：即使完成了任务，也很少系统性地总结当天、规划明天，成长是零散的。

市面上的待办类 / 番茄钟类工具能解决"记录"，但解决不了**坚持**——它们是被动的清单，不会追问你、不会因你的怠惰而"失落"。本产品的核心赌注是：**把一个有人格、有记忆、会追问、会随你进度产生情绪的虚拟角色放在桌面上，用"陪伴 + 问责"替代纯粹的提醒。**

## 2. 目标用户与画像

主用户即开发者本人（个人工具，单用户）。

- **身份**：研究生，长时间面对电脑工作。
- **场景**：每天需阅读文献并产出总结/反思；并行多门课程的作业；有持续的自我提升意愿。
- **特征**：技术能力强（能自行开发与维护）；偏好动漫文化（希望角色是喜欢的二次元形象）；对文学性内容有审美需求。
- **核心诉求**：被温和而坚定地"盯着"完成每天该做的事，并在一天结束时被引导复盘。

## 3. 产品目标与成功指标

**北极星指标**：连续使用天数（streak）——它同时反映了"产品被打开"和"习惯被建立"。

| 维度 | 指标 | 目标（V1 后一个月） |
|---|---|---|
| 习惯养成 | 连续每日总结天数 | ≥ 14 天不断 |
| 行为改变 | 任务记录后的实际完成率 | 较使用前主观提升 |
| 复盘闭环 | 晚间总结完成率（弹出后真的写了） | ≥ 70% |
| 主观价值 | "忘记任务"的发生频率 | 明显下降 |

> 注：个人工具，不追求 DAU / 留存 / 收入，成功 = 真的改变了使用者的日常行为。

## 4. 设计原则（产品理念）

1. **问责优先于提醒**：核心不是"弹个通知"，而是"它记得你昨天的承诺，今晚会拿出来对质"。
2. **三层解耦**：数据层（骨）→ 智能层（脑）→ 形象层（皮），可独立开发、独立验证，按此顺序建设。
3. **形象是动力，不是核心**：Live2D 角色负责提供陪伴感与"不忍辜负"的情绪驱动，但不能喧宾夺主。
4. **温和而坚定**：人格设定避免说教与压迫；用文学性、共情的语气表达关切。
5. **数据本地、隐私自持**：所有任务与日志存在本地，用户完全掌控。

## 5. 功能需求

优先级：**P0 = MVP 必须**，**P1 = V1 重要**，**P2 = 后续增强**。

### 5.1 任务管理（骨）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-1.1 | 添加任务（标题、详情、来源、截止日、优先级） | P0 | 能录入并持久化，重启后仍在 |
| F-1.2 | 查看"今日任务"（未完成且截止日≤今天） | P0 | 按优先级与截止日排序展示 |
| F-1.3 | 标记完成 / 放弃（保留放弃痕迹，不物理删除） | P0 | 状态变更被记录，可供晚间追问 |
| F-1.4 | 任务来源分类（课程作业 / 文献 / 个人…） | P1 | 可按来源筛选 |
| F-1.5 | 截止日临近 / 逾期高亮提醒 | P1 | 逾期任务在提醒中被特别标注 |

### 5.2 形象与桌面交互（皮）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-2.1 | Live2D 角色显示于桌面（透明、置顶、可拖动） | P1 | 模型正常渲染，窗口透明无白边 |
| F-2.2 | 点击穿透 / 可切换（不挡操作） | P1 | 角色不影响正常使用桌面 |
| F-2.3 | 待机动作 + 表情，随状态变化 | P2 | 完成任务时"开心"，拖延时"失落" |
| F-2.4 | 对话气泡显示文字 | P1 | 问候/佳句/追问以气泡呈现 |
| F-2.5 | 口型同步 + 语音（TTS） | P2 | 可选开关，默认关 |
| F-2.6 | 角色视觉配置（选用的 Live2D 模型、名字、对你的称呼） | P1 | 改配置即换形象与称呼，不改代码 |

> MVP 阶段可先用一张静态立绘/GIF 占位，把功能跑通后再上完整 Live2D。

### 5.3 文学性内容（"佳句"）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-3.1 | 不定时弹出佳句，来源为「现成语料库 + AI 生成」混合 | P1 | 频率可配置；语料库提供基底，AI 提供应景句，不重复刷屏 |
| F-3.2 | 佳句与当天主题/心情挂钩（AI 生成/改写） | P2 | 能结合当日总结生成应景句子 |

> 语料库可由你自建（诗词、名言、个人摘抄）——正好用上你的语料处理能力；AI 负责按当日心情挑选与改写。

### 5.4 每日总结与反思（脑 × 骨）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-4.1 | 每晚定时提醒写总结 | P0 | 到点弹出输入入口 |
| F-4.2 | 逐项追问任务完成情况并记录 | P0 | 对今日每个任务确认 完成/未完成/原因 |
| F-4.3 | 引导式反思（角色提问，用户作答） | P1 | 至少一轮共情式追问 |
| F-4.4 | 总结与反思持久化，可回看历史 | P0 | 按日期可查任意一天 |

### 5.5 智能分析与规划（脑）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-5.1 | 早/午/晚三段问候，按当前时段自动选择，启动时即触发 | P0 | 开机或每日首次启动按当前时段问好：早安播报今日任务、午安轻量关心、晚安引出总结 |
| F-5.2 | 基于完成情况 + 当日心得，生成分析 | P0 | 输出对今日的简要点评 |
| F-5.3 | 规划明日行动并写回当天日志 | P0 | 生成可执行的明日清单 |
| F-5.4 | 周期性回顾（近 7 天趋势 / 反复拖延的任务） | P2 | 能识别长期模式并反馈 |

### 5.6 养成与问责系统（脑 × 皮）

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-6.1 | 连续天数 streak 统计与展示 | P1 | 断更会清零并被角色"惋惜" |
| F-6.2 | 角色情绪随完成度变化 | P2 | 高完成→积极；连续拖延→低落 |
| F-6.3 | "昨日承诺"对质（拿昨晚规划追问今天） | P1 | 晚间总结时引用昨日 plan_next |

### 5.7 设置与个性化

| 编号 | 功能 | 优先级 | 验收标准 |
|---|---|---|---|
| F-7.1 | 自定义各时段时间（早/午/晚问候、晚间总结） | P0 | 用户在设置中调整，调度按新值生效 |
| F-7.2 | 选择角色人格预设（三选一），可随时切换 | P0 | 选定后全局语气一致，切换即时生效 |
| F-7.3 | 佳句弹出频率与开关 | P1 | 可调频率、可关闭 |
| F-7.4 | 自定义角色名字与对你的称呼 | P1 | 改后在所有对话中体现 |

**三种人格预设**（决定大脑层所有 prompt 的语气，是产品的"声音"，属跨模块设定）：
- **严师型**：直接、目标导向、督促严格，强调纪律与截止日。
- **温柔陪伴型**：共情、鼓励、温暖，重在情绪支持与肯定。
- **毒舌挚友型**：调侃犀利、反向激励，刀子嘴豆腐心，关心藏在吐槽里。

## 6. 关键用户旅程

**早晨（启动）**：应用随开机启动 → 角色按当前时段问好 → 早安播报今日任务（含逾期高亮）→ 给出第一句佳句。

**中午（午安）**：角色送上午安问候 + 一句佳句，轻量关心进度 → 不定时弹出佳句 → 截止日临近的任务主动提醒。

**夜晚（复盘）**：到设定时间，角色提醒写总结 → 逐项追问今日任务完成情况并记录 → 一轮引导式反思 → AI 综合完成度与心得，给出今日点评 + 明日规划 → streak 更新，角色根据结果表达情绪。

## 7. 非功能需求

- **性能**：桌面常驻，CPU/内存占用低，Live2D 渲染稳定（注意纯 Python 渲染性能上限）。
- **可靠性**：定时任务不漏触发；崩溃不丢数据。
- **隐私**：任务与日志仅存本地；调用云端 LLM 时仅传必要内容，敏感信息可脱敏。
- **可维护性**：LLM 通过抽象层接入，可在 DeepSeek API 与本地 Ollama 间切换，不改业务逻辑。
- **启动**：支持开机自启（启动文件夹 / 任务计划程序）。

## 8. 技术方案概览

- **形态**：Python 自研桌面应用（从零搭建，掌控全部架构）。
- **界面/形象**：PySide6 透明置顶窗口 + `live2d-py`（C 扩展封装 Live2D Native SDK，经 OpenGL 直接渲染）。
- **数据层**：标准库 `sqlite3`，本地数据库置于用户目录。
- **大脑**：DeepSeek API（中文与文学性强、指令遵循好、成本低、OpenAI 兼容），经 `brain/llm.py` 抽象，未来可换本地 Ollama。
- **调度**：APScheduler 负责早安/提醒/晚间总结的定时触发。
- **通知**：windows-toasts / 系统通知作为气泡之外的补充提醒。

模块划分：`core/`（数据+调度）→ `brain/`（LLM+prompt+agent 逻辑）→ `ui/`（窗口+气泡+总结输入）。

## 9. 数据模型（已实现）

- **tasks**：id、title、detail、source、due_date、priority、status（pending/done/dropped）、created_at、completed_at。
- **daily_logs**：id、log_date（唯一）、summary、reflection、mood、plan_next（AI 生成）、created_at。

> 说明：任务用 status 标记而非删除，以保留"放弃"痕迹供问责；daily_logs 支持同日增量更新（用户先写总结、AI 后补规划进同一条）。

## 10. 版本规划（Roadmap）

| 版本 | 目标 | 主要范围 |
|---|---|---|
| **MVP** | 验证核心价值闭环 | 最简图形界面 + F-1.1~1.3、F-4.1~4.2、F-4.4、F-5.1~5.3、F-7.1~7.2（含三段问候与人格选择，暂无 Live2D 形象）|
| **V1** | 完整体验 | 上 Live2D（F-2.x）、佳句（F-3.1）、引导反思（F-4.3）、streak 与对质（F-6.1/6.3）、开机自启 |
| **V2** | 智能增强 | 周期性回顾（F-5.4）、情绪系统（F-6.2）、应景佳句（F-3.2）、语音 TTS（F-2.5）|

**MVP 切线原则**：先不做任何形象，把"记录任务 → 晚间追问 → AI 分析规划"这条闭环跑通并形成习惯，再投入 Live2D。

## 11. 范围外（本期不做）

- 多用户 / 云同步 / 移动端。
- 与第三方日历、待办、邮箱的集成（未来可考虑）。
- 角色的复杂养成玩法（喂养、装扮、商店等游戏化系统）。

## 12. 风险与对策

| 风险 | 影响 | 对策 |
|---|---|---|
| 提醒被习惯性无视 | 产品沦为又一个被忽略的闹钟 | 强化"问责/对质/情绪"机制，而非单纯通知 |
| Live2D 纯 Python 渲染性能不足 | 卡顿、占用高 | MVP 先静态立绘；性能不达标时降级方案 |
| 习惯中断后放弃使用 | streak 清零打击积极性 | 角色以共情而非指责回应断更，提供"重新开始"引导 |
| LLM 成本/不稳定 | 体验受影响 | 抽象层支持切换本地模型；非关键调用可缓存/降频 |
| 过度依赖 AI 规划 | 失去自主性 | AI 只给建议，最终由用户确认 |

## 13. 决策记录（v0.2 已敲定）

| # | 问题 | 决策 |
|---|---|---|
| 1 | 各时段时间是否可自定义 | 是，用户自定义（早/午/晚问候 + 晚间总结时间）|
| 2 | 问候触发方式 | 启动即按当前时段触发；早、午、晚三段问候都做 |
| 3 | 佳句来源 | 现成语料库 + AI 生成 混合 |
| 4 | 角色人格 | 提供三种预设（严师 / 温柔陪伴 / 毒舌挚友），由用户选择 |
| 5 | MVP 界面 | 提供最简图形界面（非纯命令行）|

### 新浮现的待决问题（v0.3 再定）

1. "晚安"问候与"晚间总结"是同一个入口，还是分开（先晚安、总结另起）？
2. 最简 GUI 的形态：常驻小窗 / 系统托盘图标 + 弹窗 / 单一主窗口？
3. 人格预设是否同时影响"情绪系统"的表现（如毒舌型用调侃来表达失落）？
4. 自建佳句语料库的来源与规模（诗词 / 名言 / 个人摘抄）？
