# 第四阶段：连续复盘与昨日承诺

这一阶段实现 PRD 的两项 V1 问责能力：

- F-6.1：连续每日总结天数（streak）统计与展示；
- F-6.3：在晚间复盘时明确展示昨天写下的 `plan_next`。

## 连续天数规则

只有写了非空“今日总结”的日期才计入连续复盘。

- 今天已经复盘：从今天向前连续计算；
- 今天尚未复盘：允许从昨天向前计算，白天打开应用不会提前断 streak；
- 昨天也没有复盘：视为已经中断，显示 0 天。

## 昨日承诺

复盘页会在今日任务前展示昨天生成的明日计划。它不是新的 AI 请求，而是从本地
SQLite 日志读取，因此离线时也始终可见。
